memory Key-Value Store driver¶

The memory driver stores key-value pairs in a transient, in-memory table. It is useful for manipulating data in memory and for testing. It includes full support for multi-key transactions.

memory Key-Value Store JSON Schema¶

object with members:

driver
Required
“memory”
context
Context

Specifies context resources that augment/override the parent context.
memory_key_value_store
Context resource

Specifies or references a previously defined memory_key_value_store context resource.
atomic
boolean (default is true)

Support atomic multi-key transactions. If set to false, the transaction behavior matches that of the file Key-Value Store and gcs Key-Value Store drivers, which may be useful for testing purposes.

memory_key_value_store context resource JSON Schema¶

object

Represents the in-memory key-value store. If multiple memory Key-Value Store specifications reference the same memory_key_value_store context resource, they all refer to the same in-memory set of key/value pairs.
